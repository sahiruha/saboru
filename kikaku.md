# サボロー 企画書

## 1. 企画タイトル

**サボロー**

〜タスクを早く終わらせる時代に、あえて“先延ばし力”を競うアプリ〜

## 2. 一言でいうと

サボローは、AIがタスクを整理・定量化し、ユーザーがどれだけギリギリまで先延ばしできたかをスコア化して競う、**人をダメにするタスク管理アプリ**です。

ただし、裏側では「人がどんなタスクを、なぜ先延ばしするのか」という本音データを蓄積し、最終的にはユーザー自身の**タスクに対する取扱説明書**を作ることを目指します。

## 3. 背景

AIエージェントの普及によって、タスクは今後どんどん整理され、分解され、実行可能な状態に変換されていきます。

しかし、その結果として人間には別の問題が生まれます。

**AIによってタスクが減るどころか、逆に“やれること”が増えすぎて、仕事が増えていく。**

早く終わらせれば、また次のタスクが降ってくる。

処理能力が上がるほど、さらに処理を求められる。

そう考えると、もはや現代において重要なのは「いかに早く終わらせるか」ではなく、**いかに人間らしく、うまくサボるか**なのではないか。

サボローは、この逆張りの価値観から生まれたサービスです。

## 4. コンセプト

通常のタスク管理アプリは、タスクを早く終わらせることを正義とします。

一方、サボローでは逆です。

**タスクを早く終わらせるのは悪。ギリギリまで先延ばしして、最後に間に合わせる人こそ偉い。**

この世界観の中で、ユーザーは自分のタスクを登録し、AIに定量化され、終了タイミングに応じて「どれだけうまくサボれたか」を評価されます。

## 5. 想定ユーザー

主なターゲットは、タスクに追われている若手社会人・AIツールを日常的に使う人です。

特に以下のような人を想定しています。

- タスク管理アプリを入れても続かない人
- いつも締切ギリギリで動く人
- 「やばい、まだ何もやってない」を友達と共有したい人
- AIによって仕事が早くなったはずなのに、逆にタスクが増えている人
- 本当はサボりたいが、罪悪感を感じている人

## 6. サービス体験

サボローの基本体験は、以下の流れです。

```
1. AI：自動でタスクの整理＆タスク生成
↓
2. AI：タスク定量化
↓
3. ユーザー：承認 or ブラッシュアップ
↓
4. ユーザー：タスク登録（候補の中からの選択）
↓
5. AI：先延ばし支援（やらない言い訳を生成）→本音データを蓄積
↓↑
6．ユーザー：先延ばしを許容 or 許容しない
↓
7. ユーザー：タスク終了
↓
8. AI：タスクジャッジ
↓
9. AI：週次総合得点化
↓
10. システム：ランキング・共有
```

## 7. 全体フローと歩留まり設計

サボローでは、単に機能を並べるのではなく、ユーザーがどこで離脱するかを意識して設計します。

### Step 1：タスク整理＆生成エージェント

ユーザーが手入力するだけでなく、将来的にはSlack、Discord、Google Calendar、Notion、メールなどから、AIが「これ、あなたのタスクですよね？」と自動で拾ってくる状態を目指します。

MVPでは、MCPから取得したことを前提に初期データを格納して始めます。

AIはそのタスクに対して、必要な情報を聞き返します。（以下の内容が埋まっていたら聞く）

```
・締切はいつ？
・関係者は誰？
・遅れたらどれくらいまずい？
・作業時間はどれくらい？
```

![スクリーンショット 2026-05-08 0.16.00.png](attachment:ca7ed856-af4e-45b0-b8a8-a7892c07ca1b:スクリーンショット_2026-05-08_0.16.00.png)

![スクリーンショット 2026-05-08 0.16.11.png](attachment:1808615f-376a-48a1-b8f7-93afb1aec12a:スクリーンショット_2026-05-08_0.16.11.png)

```
UIイメージ
・上記の内容が満たされていれば、タスクにチェックアイコン
・満たされていなければ、⚠️アイコン
→タップすると、チャット画面が表示されて、必要項目を埋めるための会話のラリーが始まる。
→埋まれば、タスクにチェックアイコン
```

ここでの歩留まりポイントは、**タスク入力の手間をどれだけ減らせるか**です。

入力が面倒だと、ユーザーは最初で離脱します。

そのため、MVPでは「一言入力 → AIが補完」という体験を重視します。

### Step 2：タスク定量化

入力されたタスクを、AIが複数の軸でスコア化します。

例：

| 評価軸 | 内容 |
| --- | --- |
| 締切の近さ | いつまでに終わらせる必要があるか |
| 重要度 | 遅れた場合の影響度 |
| 関係者数 | 巻き込む人の多さ |
| 心理的重さ | ユーザーがどれくらい嫌がっているか |
| 作業量 | 実際にかかりそうな時間 |
| 逃げやすさ | どれくらい先延ばしできそうか |

この定量化によって、今まで「なんとなく重い」と感じていた１タスクが、数値として見えるようになります。

例：

```
会議資料作成
タスクスコア：76/100点（内訳は現時点では省略）
詳細表示時に下記をレーダーチャートで表示。
タスク重さ：72点
心理的抵抗：81点
逃げやすさ：43点
危険度：68点
```

ここでの歩留まりポイントは、**定量化に納得感があるか**です。

数字が適当すぎると、ランキングにも説得力が出ません。

そのため、AIのスコアには必ず理由を添えます。

### Step 3, 4：ユーザーの承認 or ブラッシュアップと登録

タスクの承認を行うフェーズ。ここでいわゆるユーザーが理解しているタスクとAIが吐き出したタスクのミスマッチを避ける。

### Step 5：先延ばし支援

ここがサボローの特徴です。

普通のタスク管理AIなら、「早くやりましょう」と言います。

しかしサボローは違います。

サボローは、ユーザーに対して**いかにタスクを先延ばしできるか**を提案します。

例：

```
このタスクはまだ着手しなくても大丈夫だよぉ。
だって関係者Bさんの確認がまだ終わっていなくて、
あなたが今進めても手戻りになる可能性が高いからだよぉ。

おすすめサボり時間：あと4時間
```

```
ここでのサボローの人格
・怠け者
・ゆっくりとした、話口調
・話し方はめちゃくちゃおっとりなのに、言ってくる内容はめちゃくちゃ芯くってる。
```

ただの「やらなくていいよ」ではなく、文脈に基づいた“的確な先延ばし理由”を提示します。

ここでの歩留まりポイントは、**ユーザーが「ちょっと面白い」「また使いたい」と思うか**です。

サボローは便利ツールである前に、ネタとして強い体験である必要があります。

ポイント：このAIのSABOROUが言ってくるワードっていうのが、いわゆるA/Bテストみたいな問いかけになっていて、ここでの問いに対するアウトプットっていうのが、よりユーザーの取扱説明書の質を上げると思うので、それが分かるような質問を投げることを意識してほしい。

### Step 6：先延ばしを許容 or 許容しない

ここでユーザーの本音を聞き出す。

### Step ７：タスク終了

ユーザーがタスクを終えたら、アプリ上で「終了」を押します。

ここでの歩留まりポイントは、**終了報告をしたくなる設計**です。

単なる完了ボタンではなく、

```
「サボり切った」
「ギリ生還」
「まだ寝かせる」
```

のように、世界観に合わせたUXにします。

### Step 8：タスクジャッジ

タスク終了後、AIがそのタスクに対してジャッジを行います。

評価するのは、タスクを早く終わらせたかではありません。

評価するのは、以下です。

```
1タスク = タスクスコア × 先延ばし時間(締切までの%消費率)で得点を算出
```

ここでの歩留まりポイントは、**ジャッジ結果を見たくなるか**です。

AIのコメントが面白ければ、ユーザーは次のタスクも登録します。

### Step 9, 10：総合得点ランキング

各タスクのスコアをもとに、ユーザーごとの週次総合得点を算出します。（複数のタスクの合計）

スコアの考え方は以下です。

```
週次サボりスコア = 各タスクの合計
```

そのため、完全に期限を破った場合は得点無効します。

## 8. ランキング・ゲーミフィケーション

サボローでは、ランキング機能によって「サボり」を共有可能なものにします。

例：

```
今週のサボリストランキング
1位：Yutaro　総合サボりスコア 482
2位：Hiroto　総合サボりスコア 421
3位：Sato　総合サボりスコア 390
```

ランキングの種類は複数用意します。

| ランキング | 内容 |
| --- | --- |
| 今週のサボリスト | 週間総合スコア |
| ギリギリ生還賞 | 締切直前で完了したタスク |
| 重タスク放置賞 | 重いタスクを長く寝かせた人 |
| 美しい言い訳賞 | AIとの会話から生まれた名言 |
| 早く終わらせすぎたで賞 | サボロー的には負け |

これにより、タスク管理アプリでありながら、SNS的・ゲーム的な体験を持たせます。

## 9. MVPで作る範囲

MVPでは、以下に絞ります。

```
1. タスク入力
2. AIによるタスク定量化
3. 先延ばしアドバイス生成
4. タスク終了ボタン
5. AIジャッジ
6. サボりスコア算出
7. ランキング表示
```

MVPで無理に作らないものは以下です。

```
・Slack/Notion/Google Calendar連携
・MCP連携
・カメラや表情認識
・IoT連携
・完全自動タスク収集
・高度な個人最適化
```

これらは展望として見せます。

## 10. PMF後の展望

PMF後は、サボローを単なるネタアプリではなく、**自分のタスク傾向を理解するためのデータ基盤**に進化させます。

### 展望1：自動タスク収集

Slack、Discord、Notion、Google Calendar、Gmailなどと連携し、AIが自動でタスクを抽出します。

```
「このメンション、タスクですよね？」
「この会議、事前準備必要ですよね？」
「この返信、まだ返してないですよね？」
```

ユーザーはタスクを入力する手間すらなくなります。

### 展望2：自分の取扱説明書

サボローとの会話から、ユーザーの先延ばし傾向を分析します。

例：

```
・Aさんが関わるタスクは早く着手しやすい
・Bさんが関わるタスクは心理的抵抗が高い
・朝より夜の方が着手率が高い
・資料作成系タスクは先延ばししがち
・返信系タスクは小さいが溜まりやすい
```

これにより、ユーザー自身も気づいていないモチベーション構造を可視化できます。

### 展望3：外部AIとの連携

将来的には、サボローに蓄積されたデータを、ChatGPT、Claude、Notion AIなどに接続できるようにします。

そうすると、通常のAIがタスク整理をするときにも、

```
この人はBさんが関係するタスクを先延ばししがちなので、
早めに分解してあげた方がいい
```

のように、より本人に合った支援ができるようになります。

## 11. サボローの本質

表向きには、サボローは人をダメにするサービスです。

でも、プロジェクトとして見ると、サボローは人を良くする可能性を持っています。

なぜなら、人は本音を吐ける場所でしか、自分の弱さをデータ化できないからです。

普通のタスク管理アプリでは、人は「ちゃんとやります」と入力します。

でもサボローでは、「本当はやりたくない」「まだ寝かせたい」「あの人が関わっているから嫌だ」と言える。

この本音こそが、将来的にその人を理解するための最も価値あるデータになります。

## 12. まとめ

サボローは、AI時代のタスク過多に対する逆張りの提案です。

AIによって仕事が早くなる時代だからこそ、

人間には「早く終わらせる力」だけでなく、

「うまくサボる力」が必要になる。

サボローは、タスクを定量化し、先延ばしを支援し、ギリギリで完了する美しさを競うことで、これまで罪悪感だった“サボり”を、新しい働き方の文化に変えていきます。
